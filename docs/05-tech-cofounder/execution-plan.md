# 05 - 技术联创/CTO · 5年执行计划

## 总目标
加入早期创业公司作为技术联创/CTO，5年通过股权+薪资实现600万+。

---

## Year 1：找到对的船

### Q1：筛选+接触
**月1：创业公司雷达**
| 周 | 任务 | 优先级 |
|----|------|--------|
| W1 | 明确偏好：AI赛道/阶段(天使-A轮)/城市/团队规模 | 🔴 |
| W1 | 列出目标公司清单（50家）：IT桔子/36kr/创投圈 | 🔴 |
| W2 | 每天深度研究2家公司（产品/团队/融资/方向） | 🔴 |
| W2 | 找到每家公司的创始人/关键人联系方式 | 🟡 |
| W3 | 开始networking：每周约3个创始人喝咖啡 | 🔴 |
| W4 | 参加创业活动/黑客松（增加曝光） | 🟡 |

**月2：建立信任**
- W1-W2：选择3-5家最感兴趣的公司，主动提供技术建议（免费/低价咨询）
- W3-W4：做1个技术demo/PoC展示实力

**月3：决策**
- W1-W2：拿到1-2个联创offer
- W3：尽职调查（团队/市场/股权/法律）
- W4：做决定+签署协议（注意股权谈判）

### Q2：融入+证明价值
- 月4：全面了解业务，提出技术架构优化方案
- 月5：招到前3个工程师，搭建团队
- 月6：交付第一个里程碑（MVP/核心功能）

### Q3：技术护城河
- 月7-9：建立技术壁垒（专利/算法/数据）
  - 搭建可扩展的技术架构
  - 引入关键技术决策（选型/自研vs开源）
  - 建立工程文化（代码review/CI/CD/测试）

### Q4：年度复盘
- 月10-12：公司是否有增长？团队是否稳定？股权是否合理？

---

## Year 2：产品+团队（如果公司在正轨）

### 核心策略
1. **产品驱动**：确保技术直接服务于业务指标
2. **团队建设**：从3人扩到10-15人
3. **融资支持**：用技术实力帮公司融资（技术是核心卖点之一）

### 月度节奏
```
月1-3：产品核心功能完善
月4-6：技术团队扩充+流程建设
月7-9：性能/安全/可靠性建设
月10-12：年终复盘+Y3规划
```

---

## Year 3：规模化（如果公司到A轮/B轮）

### 核心策略
1. 技术团队30+人，分层管理
2. 建立技术品牌（技术博客/开源/大会分享）
3. 参与公司战略决策（不只是技术）

---

## Year 4-5：退出/收获

### 最好情况：公司上市/被收购
- 股权价值 500-2000万
- 加上4年薪资约200-400万
- 总计 700-2400万

### 中等情况：公司稳定盈利
- 持续分红+薪资
- 年收入100-200万
- 5年累计 500-1000万

### 最差情况：公司失败
- 及时止损，带走技术经验+人脉
- 转路径02（回大厂）或路径03（独立开发）

---

## 股权谈判清单

| 条款 | 建议范围 | 注意事项 |
|------|----------|----------|
| 股权比例 | 5-15%（联创级别） | 不要<3%，不值得all in |
| 行权周期 | 4年cliff 1年 | 确保cliff后有加速行权条款 |
| 薪资 | 市场价60-80% | 不要低于50%，生活不能紧巴巴 |
| 角色 | 明确CTO/CPO title | 书面确认职责范围 |
| 董事会 | 争取observer席位 | 了解公司重大决策 |
| 退出机制 | Good/bad leaver条款 | 保护自己 |
